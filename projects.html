Database Management/SQL
Designed a relational database for a consulting firm capturing details of the ongoing projects. Jan 2021 - Apr 2021

SR Consulting (SRC) is a firm that creates custom software for its clients. When a client comes to SRC about a job, the managers at SRC determine the staffing needs in terms of the number of consultants needed and the required skills of the consultants. They then locate the available consultants with the necessary skills and assign them to the project. One of the consultants is assigned to be the project manager. When the project is completed the project manager evaluates all of the other consultants who worked on the project, and all of the project consultants evaluate the project manager. A project can be subdivided into multiple projects. Technical Specifications: 1. Designed an ER diagram using Forward Engineering in MySQL workbench by analyzing the requirements to identify business rules, entities and relationship. 2. Created a database with Oracle APEX based on the logical design by writing SQL queries for creating tables and inserting records. 3. Developed SQL queries to pull out the data from the database for tracking number of projects, consultants assigned to projects, calculating the number of hours spent on particular projects, etc.

Python Projects
Loan analysis of crowd funding platform Kiva.org using Python Sept 2021 - Dec 2021

Kiva.org is a 501(c)(3) non-profit based in San Francisco, dedicated to expanding financial access by connecting underserved entrepreneurs and students with lenders through online microfinance. Borrowers submit loan applications that require funding before expiration, following approval by the organization. The goal of this project was to identify key factors influencing loan repayment and to predict whether a loan application would be funded. Logistic Regression and Decision Tree classifiers were developed, trained, and evaluated using K-fold cross-validation, classification reports, and confusion matrices. The Decision Tree model demonstrated superior performance.

Loan Analysis: Understanding The Client And Business using Python May 2020 - June 2020

A second hand dataset from Kaggle was used to analyze loan behavior using logistic regression. Machine learning model was trained to predict loan repayment. To determine the loan eligibility criteria, bank adds weightage for parameters collected from sources such as customers, credit rating agencies, to understand their impact in the final decision making process. Banks use this information to check if an applicant is eligible for loan. To get better result, two weightage list were created .Model 1 and Model 2 tables had some parameters that a bank collects, in their order of priority, along with the granted weightage and and models were developed on each of them. Process Maps were developed for both models (model 1 and model 2). Machine Learning model was evaluated via K-fold classification technique, confusion matrix and correlation matrix. Feature Importance was performed to identify the most useful features in the dataset. By comparing these different models' output , important parameter / factors for loan approval decision making process was determined. .
